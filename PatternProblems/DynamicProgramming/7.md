- Can be solved in O(nlogn) time using a binary search approach.
- But can also be solved in O(n^2) time using a dynamic programming approach.
- I used the dynamic programming approach to solve this problem.
- The idea is to create a dp array where dp[i] represents the length of the longest increasing subsequence that ends with the element at index i.
- We initialize the dp array with 1, since the minimum length of an increasing subsequence is 1 (the element itself).
- We then iterate through the array and for each element, we check all the previous elements to see if we can form an increasing subsequence.
- If the current element is greater than the previous element, we can update the dp[i] value to be the maximum of its current value and dp[j] + 1, where j is the index of the previous element.
- Finally, we return the maximum value in the dp array, which will give us the length of the longest increasing subsequence.
